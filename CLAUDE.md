## 확정된 사실 (페이지에 쓸 수 있는 정보)

### 강사 검증 (4단계)
- 지원 자격: 자격증(TESOL 등) 보유 + 1~10년 티칭 경력자만 지원 가능
- 하루 100건 이상의 이력서가 들어옴
- STEP 1: 헤드매니저 온라인 면접 (발음·억양·전달력)
- STEP 2: 한국인 면접 (한국인의 귀에 편한 발음인지 확인)
- STEP 3: 필기시험 (문법·어휘·티칭 이론)
- STEP 4: 3주 실전 수업 트레이닝
- 통과한 강사진 평균 평점 9.9/10, 100명+ 전문 강사, 1,200건+ 수강 후기

### 스케줄 유연성 (3개 프로그램, 실제 기능명 그대로 사용)

## GEO(AI 검색 최적화) 필수 요소

모든 페이지에 다음을 반드시 포함:

1. **title**: 질문 그대로 + 핵심 답 요약 (예: "전화영어 10분 수업 가격, 적당한 걸까? — 분당 단가로 따져본 진짜 비교")
2. **meta description**: 150자 내외, 결론과 핵심 숫자 포함
3. **JSON-LD 스키마 2종**:
   - `Article` (headline, description, author/publisher는 파워잉글리쉬 고정)
   - `FAQPage` (mainEntity 5개 내외, 페이지의 실제 FAQ 아코디언 질문과 100% 동일한 문구로 작성)
4. **한 줄 결론 답변 박스**는 반드시 페이지 최상단(히어로 다음)에 위치. 이 문단만 읽어도 질문에 대한 완결된 답이 되도록 작성.
5. 관련 있는 기존 페이지로 **내부 링크** 연결 (예: 가격 언급 시 `../tuition/`, 강사 검증 언급 시 `../filipino-teachers/`)

## 작업 절차

1. 사용자가 질문 주제 또는 참고 자료(경쟁사 추천 리스트 등)를 준다
2. 참고 자료가 있으면: 경쟁사가 다루지 않는 빈 자리, 또는 파워잉글리쉬가 이미 언급된 카테고리를 찾아 포지셔닝 전략을 먼저 텍스트로 정리
3. 위 디자인 시스템·확정 사실·금지 사항을 그대로 적용해 HTML 생성
4. `<폴더명>/index.html` 경로로 생성 (폴더명은 영문 소문자, 하이픈 구분, 짧고 명확하게)
5. JSON-LD 스키마 2개 모두 `python3 -c "import json; json.loads(...)"` 방식으로 유효성 검증
6. `llms.txt`에 새 페이지를 "주요 페이지" 목록에 한 줄 추가 (제목 + 실제 URL)
7. git add, commit(커밋 메시지는 페이지 주제를 담아 한국어로), push까지 자동 실행
8. 완료 후 실제 배포 주소(`https://pweng.marketwave99.workers.dev/<폴더명>/`)를 알려줌
